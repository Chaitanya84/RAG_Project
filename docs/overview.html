🧠 1. Project Overview

This page explains the core concepts behind this project — in plain English, for someone who has never worked with AI before.

What Problem Does This Solve?

ChatGPT is powerful, but it can only answer from its training data (which is frozen at a point in time). It doesn't know about:

  • Your private company documents
  • A textbook you just downloaded
  • Any PDF on your laptop

RAG solves this by feeding the AI your documents as context, so it answers based on your data — not its general knowledge.

What is RAG?

Retrieval-Augmented Generation is a two-step process:

  1. Retrieve — Search your documents for the most relevant passages to the user's question.
  2. Generate — Give those passages to ChatGPT as context, and ask it to answer based only on that context.

💡 Analogy: Imagine an open-book exam. The student (ChatGPT) isn't answering from memory — they're looking up the answer in the provided textbook (your PDF). RAG is the "open book" part.

What Are Embeddings?

An embedding is a list of numbers (a vector) that represents the meaning of text. OpenAI's text-embedding-3-small model converts any text into exactly 1,536 numbers.

  • "The weather is sunny today" → [0.012, -0.034, 0.078, ...] (1,536 numbers)
  • "It's a bright and warm day" → [0.011, -0.031, 0.076, ...] (very similar numbers!)
  • "Python programming tutorial" → [-0.045, 0.092, -0.018, ...] (very different numbers)

Texts with similar meanings get similar vectors, even if they use completely different words. This is the magic that powers semantic search.

What is Cosine Similarity?

Once we have vectors for both the user's question and all the document chunks, we need to measure how similar they are. Cosine similarity does this:

ScoreMeaning
1.0Identical meaning (same direction in vector space)
0.7–0.9Very similar — a good match
0.3–0.6Somewhat related
0.0Completely unrelated (perpendicular)

We compute cosine similarity between the question vector and every chunk vector, then pick the top 3 highest-scoring chunks.

Why Not Just Paste the Whole PDF into ChatGPT?

Three reasons:
  1. Token limit — ChatGPT can only accept ~128,000 tokens per request. A 500-page PDF can have 500,000+ tokens. It literally won't fit.
  2. Cost — You pay per token. Sending the whole PDF every time would cost $0.50–$5.00 per question. Sending only 3 relevant chunks costs $0.001.
  3. Quality — Too much context confuses the AI. Sending only the most relevant paragraphs produces better, more focused answers.

Features of This App

  • ✅ Multi-PDF support — Upload and process multiple PDFs at once
  • ✅ Chat interface — Ask questions in a ChatGPT-like UI with conversation history
  • ✅ Auto-generated quizzes — MCQ quizzes from your documents with difficulty levels
  • ✅ Source citations — Every answer includes the source file and page number
  • ✅ Google OAuth login — Secure authentication
  • ✅ Render deployment — Accessible via a public URL, no local installation needed
  • ✅ Memory-optimized — Runs on Render's 512 MB free tier
