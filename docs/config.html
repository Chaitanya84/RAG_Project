🔧 3. Configuration — config.py

This file is the control panel of the entire app. Every other file imports settings from here. If you need to change a path, a model name, or a default value, this is the only file you edit.

Why One Config File?

Without a central config, you'd have API keys and file paths scattered across 6+ files. If you change something, you'd need to find and update it everywhere. config.py solves this — single source of truth.

Section 1: App Root & .env Loading

# Where this file lives on disk
APP_ROOT = os.path.dirname(os.path.abspath(__file__))

# Read .env file from the app folder
load_dotenv(os.path.join(APP_ROOT, ".env"))
  • APP_ROOT — The absolute path to the RAG_Streamlit_App/ folder. All other paths are built relative to this.
  • load_dotenv() — Reads the .env file and makes its values available via os.getenv(). This is from the python-dotenv library.
💡 What is __file__? It's a built-in Python variable that holds the path of the current script. os.path.abspath(__file__) makes it an absolute path (e.g., /home/user/RAG_Streamlit_App/config.py), and os.path.dirname() strips the filename to get the directory.

Section 2: Logging

LOG_LEVEL = logging.DEBUG if os.getenv("DEBUG", "false").lower() == "true" else logging.INFO

logging.basicConfig(
    level=LOG_LEVEL,
    format="[%(levelname)s] %(name)s – %(message)s",
)
logger = logging.getLogger("rag_app")
  • If DEBUG=true in your .env, you get very detailed logs. Otherwise, only important messages.
  • logger is used throughout all other files via from config import logger.
💡 What is logging? Instead of using print() everywhere, Python's logging module lets you categorize messages by severity: DEBUG (detailed), INFO (normal), WARNING, ERROR. You can turn verbosity up or down with one setting.

Section 3: OpenAI Settings

OPENAI_API_KEY = os.getenv("OPENAI_API_KEY", "")
OPENAI_EMBEDDING_MODEL = os.getenv("OPENAI_EMBEDDING_MODEL", "text-embedding-3-small")

Section 5: Chunking Defaults

NUM_SENTENCE_CHUNK_SIZE = 10   # Group every 10 sentences into one chunk
MIN_TOKEN_LENGTH = 30          # Skip chunks shorter than ~120 characters
PAGE_NUMBER_OFFSET = 0         # Subtract from raw page numbers (for front-matter)
🎯 Why 10 sentences per chunk? Too small (1–2 sentences) = not enough context for a good answer. Too large (50+ sentences) = too much noise, wastes tokens. 10 is a good balance — each chunk is roughly a paragraph.

Section 6: OAuth Settings

DEBUG_MODE = os.getenv("DEBUG", "false").lower() == "true"

if DEBUG_MODE:
    os.environ["OAUTHLIB_INSECURE_TRANSPORT"] = "1"

REDIRECT_URI = os.getenv("REDIRECT_URI", "http://localhost:8501/")
CLIENT_SECRET_PATH = os.path.join(APP_ROOT, "client_secret.json")
  • OAUTHLIB_INSECURE_TRANSPORT=1 — Allows OAuth over HTTP (not HTTPS). Only for local development! In production (Render), the connection is HTTPS so this isn't set.
  • REDIRECT_URI — Where Google sends the user back after login. Locally it's localhost:8501; on Render it's your deployed URL.
  • If client_secret.json doesn't exist on disk (e.g., on Render), the app creates it from the GOOGLE_CLIENT_SECRET_JSON environment variable.

Section 7: OpenAI Client (Lazy Singleton)

_openai_client: OpenAI | None = None

def get_openai_client() -> OpenAI:
    global _openai_client
    if _openai_client is None:
        _openai_client = OpenAI(api_key=OPENAI_API_KEY)
    return _openai_client
💡 What is a Singleton? A design pattern where only ONE instance of something is created. Here, we create the OpenAI client the first time it's needed, then reuse it forever. This avoids creating a new connection for every API call.

Lazy means it's not created when the file is imported — only when get_openai_client() is actually called. This saves memory if the client is never needed (e.g., during page loads that don't make API calls).

Section 8: Security

MAX_QUERY_LENGTH = 1000

Limits user questions to 1,000 characters. This is a basic defense against prompt injection — where a malicious user sends a very long prompt to trick the AI or waste your OpenAI credits.

How Other Files Use config.py

# In embedandchunk.py, AnswerAll.py, app.py, etc.:
from config import (
    OPENAI_API_KEY,
    OPENAI_EMBEDDING_MODEL,
    DATA_DIR,
    get_openai_client,
    logger,
)

Every file imports what it needs from config. Nobody hard-codes paths or API keys.
