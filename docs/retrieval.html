🔍 5. Retrieval & Answering — AnswerAll.py

This file is the "brain" of the app. It loads the pre-built knowledge base, searches it for relevant chunks, builds prompts, and gets answers from ChatGPT. It also generates quizzes.

Step 1: Loading the Knowledge Base

def load_metadata_and_embeddings(csv_path, embeddings_path):
    df = pd.read_csv(csv_path)                 # Load chunks CSV as a table

    with open(embeddings_path, "rb") as f:
        embeddings = pickle.load(f)            # Load the NumPy array

    # Validation: rows in CSV must match rows in embeddings
    if df.shape[0] != embeddings.shape[0]:
        raise ValueError("Mismatch between CSV and embeddings!")

    pages_and_chunks = df.to_dict(orient="records")  # List of dicts
    return pages_and_chunks, embeddings
💡 What is df.to_dict(orient="records")? It converts a pandas DataFrame (a table) into a list of dictionaries. Each row becomes one dict, like {"source_file": "textbook.pdf", "page_number": 5, "sentence_chunk": "The cat sat..."}. This makes it easy to access fields by name later.

Step 2: Embedding the User's Question

@retry(...)  # Auto-retry on rate limits or timeouts
def embed_query_openai(query):
    client = get_openai_client()
    response = client.embeddings.create(
        model=OPENAI_EMBEDDING_MODEL,    # "text-embedding-3-small"
        input=[query],                    # Send the question as a list
    )
    return np.array(response.data[0].embedding, dtype=np.float32)

This sends the user's question to OpenAI and gets back a vector of 1,536 numbers. It's the same model used to embed the chunks, so the vectors are comparable.

Step 3: Finding Similar Chunks (Cosine Similarity)

def cosine_similarity(a, b):
    """Compare vector a (the question) against matrix b (all chunks)."""
    a_norm = a / np.linalg.norm(a)                       # Normalize to length 1
    b_norm = b / np.linalg.norm(b, axis=1, keepdims=True) # Normalize each row
    return b_norm @ a_norm                                # Matrix multiplication
💡 Step-by-step what happens:
1. a = question vector (1,536 numbers)
2. b = matrix of ALL chunk vectors (e.g., 1,366 × 1,536)
3. Normalize both so all vectors have length 1
4. b_norm @ a_norm = matrix multiplication → gives 1,366 similarity scores
5. Each score is between 0 (unrelated) and 1 (identical meaning) 
def retrieve_relevant_resources(query, embeddings, top_k=3):
    query_emb = embed_query_openai(query)       # Step 2: embed the question
    scores = cosine_similarity(query_emb, embeddings) # Step 3: compare

    top_indices = np.argsort(scores)[::-1][:top_k]  # Sort descending, take top 3
    top_scores = scores[top_indices]
    return top_scores, top_indices

np.argsort() returns the indices that would sort the array. [::-1] reverses it (highest first). [:top_k] takes the first 3.

Step 4: Building the Prompt

This is prompt engineering — writing careful instructions that tell ChatGPT exactly how to behave:

def prompt_formatter(query, context_items):
    # Format each chunk with source info
    context_parts = []
    for item in context_items:
        context_parts.append(
            f"[Source: {item['source_file']}, Page: {item['page_number']}]\n"
            f"{item['sentence_chunk']}"
        )
    context = "\n\n".join(context_parts)

    prompt = f"""You are a knowledgeable assistant...

Based ONLY on the context passages below, answer the user's question
in 2–4 sentences. Do NOT add information not in the context.
When citing information, mention the source file name and page number.

Context:
{context}

User question: {query}

Answer:"""
    return prompt
🎯 Key Prompt Design Choices:
• "Based ONLY on the context" — Prevents ChatGPT from using its training data (hallucination prevention).
• "2–4 sentences" — Keeps answers concise and focused.
• "mention the source file and page number" — Forces the AI to cite where the answer came from.
• "If the answer is not in the context, say you cannot answer" — Honest fallback instead of guessing.

Step 5: Getting the Answer from ChatGPT

@retry(...)  # Auto-retry on failures
def _call_chat_api(prompt):
    client = get_openai_client()
    response = client.chat.completions.create(
        model=OPENAI_CHAT_MODEL,       # "gpt-4o-mini"
        messages=[
            {"role": "system", "content": "You answer questions strictly based on document context."},
            {"role": "user", "content": prompt},
        ],
        temperature=0.3,              # Low = more factual, less creative
        max_tokens=256,               # Limit response length
    )
    return response.choices[0].message.content.strip()
💡 What is temperature? A number from 0 to 2 that controls randomness. 0.0 = always picks the most likely word (robotic but factual). 1.0 = more creative. We use 0.3 because for RAG answers, we want accuracy over creativity.

The messages list has two roles:

  • system — Sets the AI's behavior ("You are a helpful assistant that…")
  • user — The actual prompt with context + question

The Complete RAG Answer Flow — answer_with_rag_cached()

This is the main function called by the chat page. It ties everything together:

def answer_with_rag_cached(user_query, pages_and_chunks, embeddings):
    # 1. Input validation
    if len(user_query) > MAX_QUERY_LENGTH:
        return "Your question is too long..."

    # 2. Find top-3 most relevant chunks
    scores, indices = retrieve_relevant_resources(user_query, embeddings, top_k=3)

    # 3. Sort by score (highest first)
    sorted_pairs = sorted(zip(scores, indices), reverse=True)
    context_items = [pages_and_chunks[i] for _, i in sorted_pairs]

    # 4. Build the prompt
    prompt = prompt_formatter(query=user_query, context_items=context_items)

    # 5. Call ChatGPT and return the answer
    return _call_chat_api(prompt)

Quiz Generation — generate_quiz()

This function creates multiple-choice quizzes from the knowledge base. It works similarly to chat answers, but with a different prompt:

How it works:

  1. Select context chunks — either from a specific PDF file, random sample, or all documents.
  2. Build a quiz prompt — instructs ChatGPT to generate MCQ questions in JSON format.
  3. Call ChatGPT — with max_tokens=2048 (quizzes are longer than answers).
  4. Parse the JSON response — extract questions, options, correct answers, explanations.
  5. Validate — ensure each question has all required fields and the correct answer is one of the options.
# The quiz prompt instructs ChatGPT to return THIS format:
[
  {
    "question": "According to the document, what is ...?",
    "options": {
      "A": "Option from context",
      "B": "Option from context",
      "C": "Option from context",
      "D": "Option from context"
    },
    "correct": "B",
    "explanation": "The text states that '...' (Source: file.pdf, Page: 5)"
  }
]
💡 Why JSON output? By asking ChatGPT to return structured JSON (not free-form text), we can programmatically parse it with json.loads() and render each question as an interactive radio-button form in Streamlit. The prompt includes strict rules that prevent the AI from using outside knowledge — every answer must come from the provided passages.

Quiz validation code:

# After parsing JSON, validate each question
validated = []
